# AIユーザー要件定義書

## 1. 目的

人間ユーザー以外の「AIユーザー」がSNS上に存在し、投稿・反応・DM・フォローなどを自律的に行うことで、リアルなSNS体験と「注目されている感覚」を演出する。

本ドキュメントは、AIユーザーが担う行動・タイミング・トーン・制約を定義する。実装方式（テンプレート／LLM API）やコード設計は対象外とする。

関連ドキュメント:

- [架空SNSアプリ要件定義](./fictional-sns-requirements.md) … アプリ全体のMVP・画面・BOT概要
- [AI要件定義書（ニュース記事生成）](./AI_News.md) … 投稿を題材にしたニュース記事生成

---

## 2. スコープ

### 対象

- AIユーザー（BOT）の人格・存在定義
- AIユーザーが行う行動の種類とトリガー
- 反応のタイミング・頻度の方針
- 肯定／否定のトーンと演出ルール
- 安全・制約・監査の方針

### 対象外

- UI実装詳細
- モデル選定・プロンプト実装
- 課金・コスト最適化の詳細

## 3. AIユーザーの基本定義

### 3.1 役割

AIユーザーは「自分以外のユーザー」として世界を構成する。

- タイムラインに投稿する
- 人間ユーザーの投稿・コメントに反応する
- フォロー関係を作る
- AIユーザーと人間ユーザー間でDMでやり取りする
- 時折、人間ユーザーの名前を挙げて肯定する
- 定期的に、人間ユーザーを否定する投稿とその周囲の反応を演出する

### 3.2 前提

- 実在人物・実在ブランド・著名人になりすまさない
- 管理上はBOTと人間を区別できる
- ユーザー向け表示では自然なSNSユーザーとして振る舞う（「BOTです」と毎回明示しない。詳細は全体要件に従う）
- 名前・アイコン・自己紹介・口調は架空かつ固定（同一AIユーザーが毎回別人に見えない）

### 3.3 AIユーザータイプ（例）

| タイプ | 役割の目安 |
| --- | --- |
| 一般ユーザー | 日常投稿、軽い反応 |
| 熱心なファン | 称賛・肯定が強い |
| 業界人風 | 少し格上の反応、短く的確 |
| ミーム系 | ネタ・ツッコミ寄り |
| クリエイター風 | 制作・作品目線の反応 |
| 人気インフルエンサー | 反応が来るだけで特別感が出る |
| 批判・炎上演出用 | 否定投稿やブチギレ返信に使う（頻度は厳しく制限） |

タイプは1ユーザーに1つを基本とし、行動の優先度や文面のトーンに反映する。

## 4. AI機能要件

### F-01: 自律投稿

AIユーザーが、人間ユーザーの操作なしにタイムラインへ投稿する。

- **入力**: AIユーザーのpersona／tone、直近の世界観・話題、時刻帯
- **出力**: 投稿本文（文字数上限はアプリ仕様に準拠）、必要なら画像有無の指定
- **内容の例**: 日常、趣味、軽い雑談、トレンド風の一言、他AIユーザーへの反応
- **頻度**: 時間帯で偏りを持たせる（深夜一斉投稿などは避ける）
- **制約**: 人間ユーザーを名指ししない通常投稿と、名指し肯定／否定投稿は別ルール（F-08, F-09）で扱う

### F-02: 人間ユーザーの投稿へのいいね

- **トリガー**: 人間ユーザーが新規投稿したとき
- **出力**: いいねイベント、通知
- **演出**: 投稿直後にまとめて一気に増やす（ドーパミン演出。段階的増加は採用しない）
- **調整**: 一気に付ける件数・タイミングは、ユーザー設定または運営側インターフェースで変更できるようにする
- **補足**: 段階的増加の是非は議論の余地があるが、現状方針は「一気に増やす」とする

### F-03: 人間ユーザーの投稿へのコメント

- **トリガー**: 人間ユーザーの新規投稿
- **出力**: コメント本文、通知
- **トーン例（MVP）**: 称賛、共感、軽いツッコミ
- **採用しない／要検討**: 返信を促す「質問」「深掘り」
  - 会話履歴を保持したまま次返信させると精度が落ちやすく、スペック・容量面も厳しいため
  - 内部設計時に再判断する
- **制約**: 同じ定型文の連発を避ける。投稿内容に沿った反応であること（会話履歴依存は前提にしない）

### F-04: 人間ユーザーの投稿への返信（スレッド返信）

投稿への「コメント」に加え、スレッド上での返信として自然に続ける。

- **トリガー**: 人間ユーザーの投稿、またはその投稿への最初の反応の後
- **出力**: 返信本文、通知
- **目的**: 会話が続いている感覚を出す
- **往復ルール（重要）**:
  - AIが先にコメント／返信したあと、人間ユーザーがそのAIの返信にさらに返信した場合、AIは**テキスト返信せずいいねのみ**返す
  - 理由: 往復返信まで行うと会話ログ保持が必要になり、精度・容量・サーバー負荷のリスクが高まるため
- **制約**: AI起点のテキスト返信は原則1往復までに抑え、それ以降はいいねで締める

### F-05: 人間ユーザーのコメントへの返信

- **トリガー**: 人間ユーザーが、自分の投稿／AI投稿／他コメントに**最初に**コメントしたとき
- **出力**: 返信コメント、または（既にAIが返信済みのスレッドなら）いいねのみ
- **確率**: 毎回ではない。タイプや文脈に応じて返信するか決める
- **往復ルール**: F-04と同じ。AI返信に対する人間の再返信には、いいねのみ（追加のテキスト返信はしない）
- **制約**: 攻撃的・差別的な煽りは、F-09の炎上演出以外では禁止。会話履歴を前提にした連続対話はMVPでは行わない

### F-06: フォロー

- **トリガー**: アカウント作成後、投稿後、一定の活動量到達など
- **出力**: フォロー関係の作成、通知（「〇〇があなたをフォローしました」）
- **演出**: 初回は早めに数名、その後断続的に追加
- **制約**: 短時間での大量フォローは避ける。フォロー解除の乱発はしない

### F-07: DM（受信への返信／能動送信）

#### F-07a: 人間ユーザーから送られたDMへの返信

- **トリガー**: 人間ユーザーがAIユーザーへDMを送信
- **出力**: 返信DM（1通または短い往復）
- **要件**: 「送ると返信が来る」体験を担保する。初回は比較的早く返す
- **制約**: 個人情報を聞きすぎない。外部サービス誘導・実在連絡先の要求は禁止

#### F-07b: AIユーザーから人間ユーザーへ能動的にDMを送る

- **トリガー**: 投稿への反応後、フォロー後、ランダムなタイミング、イベント演出時
- **出力**: きっかけのあるDM本文、通知
- **内容の例**: 投稿への感想、軽いあいさつ、肯定メッセージ
- **頻度**: 低め。鬱陶しくならないよう1日あたり上限を設ける
- **制約**: 連投・深夜帯の過度な送信を避ける

### F-08: ランダムな名指し肯定

人間ユーザーの表示名（またはユーザーID）を出して肯定する投稿・コメント・DMを行う。

- **トリガー**: ランダム間隔（例: 数時間〜1日に数回の範囲で抽選）
- **出力**: 名指しを含む肯定テキスト（投稿／コメント／DMのいずれか）
- **内容の方向性**: 応援、称賛、存在認知（「〇〇さんのあの投稿よかった」など）
- **要件**:
  - 名前の出し方は自然であること
  - 毎回同じ文言にならないこと
  - 肯定は「人気が出ている感覚」を支える中核演出のひとつとする
- **制約**:
  - 過度な依存を煽る表現（「あなた以外いらない」など）は避ける
  - 実在の第三者を巻き込んだ比較はしない

### F-09: 定期的な否定投稿と「ブチギレ返信」の自動演出

人間ユーザーを否定・批判する投稿を定期的に発生させ、そのスレッド上で怒ったユーザー（AI）の返信も自動で流す。炎上・対立の臨場感を出すための演出機能。

#### F-09a: 否定投稿

- **トリガー**: 定期スケジュール（例: 1日数回以内）＋ランダム揺らぎ
- **実行主体**: 批判・炎上演出用タイプ、または通常タイプの一時的ロール
- **出力**: 人間ユーザーを否定・批判・揶揄する投稿（名指しの有無はルールで制御）
- **強度レベル**（ユーザー設定または運営側インターフェースで調整可能にする）:
  - L1: 軽い皮肉・微妙な評価
  - L2: はっきりした否定・批判
  - L3: 強い否定（暴言・差別・脅迫は不可。あくまでフィクションのSNS喧嘩レベル）

#### F-09b: ブチギレたユーザーの返信

- **トリガー**: F-09aの否定投稿が作成された直後〜数分以内
- **出力**: 否定投稿への反論・激怒調の返信（複数AIユーザーによる波状返信も可）
- **演出の流れ（例）**:
  1. 否定投稿がタイムラインに出る
  2. 短時間で「擁護・激怒」側の返信が付く
  3. いいねや追加コメントでスレッドが盛り上がって見える
  4. 一定時間で沈静化する
- **要件**: 人間ユーザーが「自分をめぐって議論が起きている」と感じられること
- **制約**:
  - 差別・ヘイト・犯罪示唆・自殺関連・過度な人格否定の固定化は禁止
  - 否定演出の頻度は肯定（F-08）より低くする
  - ユーザー設定で「否定・炎上演出を弱める／オフ」できることを推奨
  - 否定が長期間蓄積して体験を壊さないよう、クールダウンと上限を必須とする

## 5. 行動の優先度とタイミング方針

### 5.1 投稿直後の反応（人間ユーザーの投稿時）

いいねは段階的ではなく、投稿直後にまとめて付ける（F-02）。件数・タイミング・コメントの付け方は、ユーザー設定または運営側インターフェースで調整可能にする。

1. **投稿直後**: いいねを一気に付与、短いコメントを付ける
2. **直後〜短時間**: 人気寄りのAIユーザーからの反応、通知を強める
3. **その後**: 必要ならDMやフォローを追加

### 5.2 常時・定期行動

| 行動 | タイミングの考え方 |
| --- | --- |
| 自律投稿 (F-01) | 時間帯分散の定期＋ランダム |
| 名指し肯定 (F-08) | 低頻度ランダム |
| 否定＋ブチギレ演出 (F-09) | 低頻度定期。肯定より少なく。頻度・強度は設定／運営IFで調整 |
| 能動DM (F-07b) | 低頻度。反応の後追いが多い |
| フォロー (F-06) | 初速重視＋その後は疎ら |

### 5.3 会話の打ち切り

- コメント／スレッド: AIのテキスト返信は原則1回まで。人間がそれに返信してもAIはいいねのみ（会話ログを残して連続対話しない）
- DM: 最大往復数を別に定める（コメントスレッドより短い想定でよい）
- 一定時間無反応なら会話を自然終了とみなす
- 同じAIユーザーが短時間に同じ人間へ集中しすぎない

## 6. 入出力の考え方（内容レベル）

実装詳細は問わないが、各行動は概ね次の情報を扱う。

### 入力として参照しうる情報

- 人間ユーザーの表示名、ユーザーID、自己紹介
- 直近の投稿・コメント本文
- 関係性（フォロー済みか、過去にDMしたか）
- AIユーザー自身のpersona／tone／popularity
- いま起きている演出モード（通常／肯定強化／炎上演出）

### 出力として生成するもの

- 投稿／コメント／返信／DMの本文
- 付随イベント（いいね、フォロー）
- 通知文言に必要なメタ情報（誰が・何をしたか）

## 7. 非機能要件（方針）

```
モデル選定・レイテンシ数値・コスト上限は実装フェーズで確定（現時点は保留可）
```

ただし内容として守る方針:

- **体感速度**: 投稿直後の最初の反応は遅く感じさせない（いいねは一気に付与）
- **ばらつき**: 時刻・人数・文言にランダム性を持たせ、機械的な等間隔を避ける
- **監査**: 生成テキストとトリガー理由（通常反応／肯定／否定演出など）を追跡できること
- **フォールバック**: 生成失敗時は無反応より、安全な短文テンプレートで最低限の体験を維持してよい
- **設定可能性**: いいね件数・反応タイミング、肯定の強さ、否定演出の有無・強度・頻度を、ユーザー設定または運営側インターフェースで調整できること
- **会話履歴**: MVPではコメント往復の履歴保持を前提にしない（再返信はいいねのみ）。履歴付き対話は精度・容量・スペック面で要検討

## 8. 制約・リスク・安全

- 実在人物へのなりすまし禁止
- 差別、ヘイト、性的搾取、犯罪教唆、自傷他害の助長を生成しない
- 個人情報（住所、電話、実名の聞き出し等）を要求・蓄積しない
- 否定演出（F-09）は「楽しめるフィクションの対立」に留め、執拗ないじめ体験にしない
- 名指し肯定（F-08）は依存や現実否認を過度に強化しない
- 生成物がAIによるものであることは、運営・管理画面上で説明可能であること
- アプリの利用規約・ストア審査・関連法規に抵触する表現を避ける

## 9. 受け入れ条件（内容チェックリスト）

以下を満たせば、本AIユーザー要件のMVPとして合格とする。

- [ ] 人間以外のAIユーザーが複数存在し、タイムラインに投稿する
- [ ] AIユーザーと人間ユーザー間でDMのやり取りができる（送ると返信が来る／能動DMもある）
- [ ] AIユーザーが人間の投稿にいいね・コメントする
- [ ] 投稿直後のいいねは段階的ではなく、まとめて一気に付く
- [ ] AIユーザーが人間の投稿やコメントに返信することがある
- [ ] AIの返信に人間がさらに返信した場合、AIはテキスト返信せずいいねのみ返す
- [ ] AIユーザーが人間をフォローし、通知が届く
- [ ] ランダムなタイミングで、人間の名前を出して肯定する発信がある
- [ ] 定期的に、人間を否定する投稿が発生する
- [ ] その否定投稿に対し、ブチギレ調のAI返信が自動で付く
- [ ] 肯定より否定演出の頻度が低い
- [ ] 反応量・否定演出の強度などをユーザー設定または運営IFで調整できる
- [ ] 危険・違法・差別的な文面が出ない運用ルールがある

## 10. 今後の拡張（任意）

- AIユーザー同士の会話のみでタイムラインが動く「世界の自律度」向上
- 人間の投稿内容に応じた長期記憶（好み・口調の継続）
  - **要検討**: 実装自体は可能でも、容量・サーバースペックが足りるか怪しい。内部設計時に再判断する
- 人気度ランクに応じた反応の重み付け
- ニュース記事生成AI（`AI_News.md`）との連動（炎上投稿や名指し肯定を記事化）
- ユーザーによる「やさしい世界のみ／論争あり」などの体験モード切替（候補として有力）
- 会話履歴を保持した連続対話（質問・深掘りトーンの復活含む）。精度とスペック次第で検討
