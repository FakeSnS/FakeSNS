# AI本体 設計要件定義書（共通コア基盤）


> 個別機能を参照する際は、出典を **[AI_Users]** / **[AI_News]** で明記する。

---

## 1. 目的とスコープ

SNS投稿を入力に、**AIユーザーの反応生成**（`AI_Users.md`）と **ニュース記事生成**（`AI_News.md`）の両方を担う共通AIコアを設計する。
両者は「投稿を解析し、ペルソナに沿って肯定／否定のテキストを生成する」という同一構造を持つため、解析・生成・安全・監査を1つのパイプラインに束ねる。

### 対象

- 解析エンジン
- 生成エンジン
- ペルソナ／演出モード制御
- 安全フィルタ
- 監査ログ
- フォールバック

### 対象外

- UI実装
- SNSアプリ本体のイベント配信基盤

---

## 2. 全体アーキテクチャ

### 入口トリガー

| トリガー | 説明 | 主な出典機能 |
| --- | --- | --- |
| `user_post` | 人間ユーザーの新規投稿 | [AI_Users] F-02 / F-03 / F-04|
| `user_comment` | 人間ユーザーのコメント | [AI_Users] F-05 |
| `user_dm` | 人間ユーザーからのDM | [AI_Users] F-07a |
| `scheduled` | 定期・ランダム発火（自律投稿／名指し肯定／否定演出／能動DM／フォロー） | [AI_Users] F-01 / F-06 / F-07b / F-08 / F-09 |
| `news_request` | 記事化要求（投稿を題材にニュース生成） | [AI_News] F-01 / F-02 |

### 出口成果物

- `ai_reaction` … 投稿／コメント／返信／DMの本文＋付随イベント（いいね・フォロー） … **[AI_Users]**
- `news_article` … 見出し／リード／本文／カテゴリタグ／サムネキーワード … **[AI_News]**

---

## 3. 生成パイプライン詳細

### ①解析ステージ

投稿本文を解析し、News／Users 双方が再利用できる共通の解析結果を1回で生成する。

- **統合元**: [AI_News] F-01（投稿内容の解析）を共通化し、[AI_Users] の反応判断にも流用する
- **入力**: 投稿本文、投稿者名（表示名／ユーザーID）、投稿日時
- **出力**:

| 出力項目 | 内容 | 使う側 |
| --- | --- | --- |
| `topic` | トピック分類 | [AI_News]・[AI_Users] 両方 |
| `summary` | 要旨（1〜2文） | [AI_News]（リード素材）・[AI_Users]（反応の的） |
| `sentiment` | 感情極性（正／中／負＋強度） | 両方（反応トーン決定） |
| `safety_flags` | 危険カテゴリの事前検出 | 両方（③生成前ゲート） |

- **設計方針**: １回の呼び出しで(JSON)強制出力、会話履歴は入力に含めない（[AI_Users] の「MVPでは会話履歴を前提にしない」方針に準拠し、[AI_News] も単一投稿完結とする）。

### ②文脈構築ステージ

解析結果に生成用の状態を合成し、プロンプト入力を組む。
**会話履歴を持たずに「投稿に沿った反応」を成立させる**ための中核ステージ。

- **入力に含めるもの**:
  - 解析結果（①）
  - ペルソナ／トーン（[AI_Users] のpersona/tone、または [AI_News] のニュースアカウントのスタイル: オネエ／ゲイ／盲信者／ツンデレ 等）
  - 演出モード（通常／肯定強化＝[AI_Users] F-08／炎上演出＝[AI_Users] F-09）
  - 直近投稿本文（往復リプライの場合Only、当該1件のみ。スレッド全文は持たない）
- **往復ルールの強制**（[AI_Users] F-04 / F-05）:
  - AI起点のテキスト返信は原則1往復まで
  - 人間がAI返信にさらに返信した場合、生成をスキップし `like_only` を返す
  - ルールをパイプライン側で強制し、呼び出し側の実装ミスで会話ログ保持が発生しないようにする

### ③生成ステージ

成果物種別ごとにシステムプロンプトをテンプレート化し、②の文脈で生成する。

- **`ai_reaction`**（**[AI_Users]** F-01 / F-03 / F-04 / F-05 / F-07 / F-08 / F-09）:
  - 文字数はアプリ仕様に準拠
  - 定型文の連発を避けるため seed／温度でゆらぎを持たせる（[AI_Users] F-03 制約）
  - 肯定・誇張のトーンは演出モードで強度制御
  - 否定演出（[AI_Users] F-09）は強度 L1〜L3 をパラメータとしてプロンプトに反映
- **`news_article`**（**[AI_News]** F-02）:
  - 見出し30字以内／リード100字以内／本文200~600字／カテゴリタグ／サムネ用キーワード
  - 投稿元アカウントを用意されたニュースアカウントからランダム選択し、そのスタイルを注入
  - 投稿者を肯定し、誇張した表現の文体

### ④安全フィルタ（生成後ゲート）

①の `safety_flags` と二段構えで、生成後テキストを検査する。

- **共通禁止**（[AI_Users] §8 / [AI_News] §5）: 差別・ヘイト・性的搾取・犯罪教唆・自傷他害の助長、実在人物・実在ブランドへのなりすまし、個人情報（住所・電話・実名等）の要求・蓄積
- **[AI_Users] 固有**: 否定演出（F-09）が「楽しめるフィクションの対立」を超えたら差し戻し。名指し肯定（F-08）が依存・現実否認を過度に強化する表現なら差し戻し
- **[AI_News] 固有**: 著作権法32条の引用範囲を超えない。肖像権・プライバシーへの配慮。生成物が「AI生成」であることの明示
- 検出時は破棄し、⑤のフォールバックへ

### ⑤成果物整形＋フォールバック

整形して出力する。生成失敗・API障害時の挙動:

- **[AI_Users]**: 無反応より、安全な短文テンプレートで最低限の体験を維持（[AI_Users] §7）。いいね等の非テキストイベントは生成失敗と独立して発火可能にする
- **[AI_News]**: 再試行3回 → キュー退避（[AI_News] §4 フォールバック）
- **共通**: モデルバージョンを成果物メタに付与

## 4. トリガー・演出タイミング方針

パイプラインの発火制御は各出典の方針に従う。

| 行動 | 出典 | タイミングの考え方 |
| --- | --- | --- |
| いいね一気付与（段階的にしない） | [AI_Users] F-02 / §5.1 | 投稿直後にまとめて付与。件数・タイミングは設定／運営IFで調整 |
| 自律投稿 | [AI_Users] F-01 | 時間帯分散の定期＋ランダム |
| 名指し肯定 | [AI_Users] F-08 | 低頻度ランダム |
| 否定＋ブチギレ演出 | [AI_Users] F-09 | 低頻度定期。**肯定より少なく**。頻度・強度は設定／運営IFで調整。クールダウン・上限必須 |
| 能動DM | [AI_Users] F-07b | 低頻度。反応の後追いが中心。1日あたり上限あり |
| フォロー | [AI_Users] F-06 | 初速重視＋その後は疎ら |
| 記事化 | [AI_News] F-02 | `news_request` 発火時（拡張: 炎上投稿・名指し肯定を記事化） |

## 5. 監査ログ（全段記録）

両MDの監査要件のまとめ。

- **保存対象**: 入力 ／ 各ステージ出力 ／ 最終成果物 ／ モデルバージョン ／ **トリガー理由**（通常反応／肯定＝F-08／否定演出＝F-09／記事化）
- **出典**: [AI_Users] §7「生成テキストとトリガー理由を追跡できること」＋ [AI_News] §3「入力・出力・モデルバージョンを全保存（監査用）」
- 生成物がAI由来であることを運営・管理画面上で説明可能にする（[AI_Users] §8 / [AI_News] §5）

---

## 6. 本設計で確定すべき論点

両MDで「保留」だった箇所のうち、パイプライン設計上いま決めるべきもの。

- 解析（①）と生成（③）を別コールにするか1コールに畳むか（コスト vs 精度）
- ペルソナ定義のデータ構造（固定ペルソナをどこに持つか）… [AI_Users] §3.2「名前・アイコン・口調は架空かつ固定」
- クールダウン・否定演出頻度上限を、パイプライン側で持つか呼び出し側で持つか

---

## 7. 受け入れ条件（本コアとして）

- [ ] `user_post` / `user_comment` / `user_dm` / `scheduled` / `news_request` の各トリガーが1パイプラインで処理される
- [ ] 解析（①）の結果を News・Users 双方が再利用できる
- [ ] 会話履歴を保持せずに投稿に沿った反応が生成される
- [ ] 往復2回目以降は `like_only` が返る（[AI_Users] F-04 / F-05）
- [ ] 否定演出の頻度が肯定より低く、強度が設定／運営IFで調整できる（[AI_Users] F-08 / F-09）
- [ ] 記事が見出し30字・リード100字・本文200〜600字の制約を満たす（[AI_News] F-02）
- [ ] 安全フィルタが共通禁止事項を通さない（[AI_Users] §8 / [AI_News] §5）
- [ ] 全段の入出力・モデルバージョン・トリガー理由が監査ログに残る
- [ ] API障害時にフォールバック（テンプレート／再試行3回→キュー退避）が働く
